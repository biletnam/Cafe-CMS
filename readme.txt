=== О CMS ===

Cafe CMS - система управления сайтом, начинавшаяся создаваться как простенькая 
система для управления сайтом-визиткой, но в последствии выросшая в полноценную 
CMS со множеством функций. Она идеально подойдет как для создания визитки, блога
или домашней страницы, так и для небольшого портала. В нем предусмотрен все 
основные функции, которые могут потребоваться. Например, создание статичных 
страниц, блога, новостного раздела, фотоальбомов и т. п. Интуитивно понятный 
интерфейс панели управления позволит легко работать с сайтом как опытным 
разработчикам, так и простым владельцам сайтов.



=== Установка ===

1. Скачайте архив и скопируйте содержимое в корневой каталог своего сайта.

2. Откройте в браузере адрес http://ваш-сайт/install/, заполните все поля 
формы установки.


Описание полей

Настройка базы данных:
- Сервер базы данных - по-умолчанию имеет значение "localhost" и подходит для 
большинства случаев. Если ваш сервер базы данных находится на другом сервере 
(не на локальном), необходимо ввести его ip-адрес.
- Название базы данных - имя базы данных доступное пользователю.
- Имя пользователя - ваш логин для подключения к базе данных.
- Пароль - ваш логин для подключения к базе данных.
- Префикс для таблиц - по-умолчанию имеет значение "cafe"  и подходит для 
большинства случаев. Если вы устанавливаете несколько систем управления сайтом
CafeCMS, то для каждой необходимо указать разные префиксы. Префикс таблиц должен
содержать хотя бы 1 символ и не превышать 20 символов.


Регистрация администратора сайта:
- Логин - по-умолчанию имеет значение "admin" как стандартный логин 
администратора. Вы можете ввести свой логин. Логин администратора должен 
содержать хотя бы 1 символ и не превышать 20 символов.
- Пароль - придумайте и запомните пароль. Он будет использоваться для 
авторизации на сайте и в панеле управления сайтом. Пароль администратора должен 
быть не короче 6 символов.


Настройка сайта:
- Название сайта - введите название сайта. Оно будет отображаться в заголовке 
браузера, сайта и других местах как основной идентификатор. Название сайта 
не должно превышать 100 символов.


После заполнения всех полей нажмите кнопку установить. Будут созданы необходимые
таблицы в базе данных и зарегистрирован первый пользователь системы с правами
администратора. Все настройки будут записаны в файл /config.php в корневом
каталоге сайта.


3. Если установка прошла правильно, вам предложат перейти на главную страницу 
сайта или в панель управления. Установка на этом заврешена.


В целях безопасности удалите файлы папку /install со всем содержимым. 



=== Панель управления ===

Панель управления сайтом находится по адресу http://ваш-сайт/admin/. Для 
авторизации введите свой логин и пароль указанные во время установки. После 
успешной авторизации, будут доступны функции администрирования сайтом.

В правом верхнем углу отображается имя текущего пользователя.

В левой части находится основное меню, в котором перечислены все установленные 
модули, а также ссылки для перехода на сайт и выхода из панели управления (при 
этом авторизация будет сброшена).

Внизу содержится блок копирайта, ссылка на сайт разработчика Cafe CMS.

В правой (основной) части панели управления находится содержимое текущего 
модуля. Она состоит из трех частей:
- в верхней части отображаются ссылки действий текущего модуля;
- в средней части выводятся сообщения об ошибках или успешных операциях;
- в нижней части выводится содержимое текущего модуля.



=== Модули ===


=== Рабочий стол (Dashboard) ===

На данный момент этот модуль не работоспособен.
Это основной модуль, который вы видите в панели управления. В нем собрана
основная информация из всех установленных модулей.



=== Страницы ===

Модуль "Страницы" предназначен для создания статичных страниц. Доступны 
следующие функции: создание, изменение, удаление страниц, назначение позиций 
отображения в меню для каждой страницы, скрытие страницы.

Для добавления новой страницы нажмите на ссылки "Добавить новую страницу". В 
открывшейся форме заполните все поля и нажмите кнопку "сохранить".

При успешном сохранении страницы вы перейдете к списку созданных страниц. 
Заголовок страницы будет отмечен серым цветом, что символизирует не активность 
этой страниц. Также номер позиции страницы будет равен нулю. Для активации 
страницы измените это значение на отличное от нуля и нажмите кнопку "сохранить".
Это значение также будет использоваться при сортировке списка страниц.



=== Записи ===

Это универсальный раздел. Здесь вы можете создать такие разделы сайта как блог, 
новости и т. п. Перед началом работы с этим разделом необходимо создать первый 
раздел и, при необходимости, категории этого раздела. Для этого нажмите на 
ссылку "Разделы и категории", в открывшейся форме заполните поля и нажмите 
кнопку "сохранить". 

Для опубликованных на сайте записей, пользователи смогут оставлять комментарии. 
Для просмотра списка всех комментариев перейдите по ссылке "Комментарии". По 
умолчанию отображаются комментарии из всех разделов. Для отображения 
комментариев из определнного раздела отсортируйте их по разделам. Для этого 
выберите в выпадающем списке название раздела.

Для изменения или удаления комментария, нажмите соответствующие ссылки.



=== Фотографии ===

Модуль "Фотографии" позволяет размещать на сайте изображения группируя их 
в альбомы. Фотографии сохраняются в трех разрешениях: 200*200px для превью, 
800*600px для просмотра и в оригинальном разрешении для дальнейшей обработки 
или скачивания.


 
=== Пользователи ===

Управление пользователями осуществляется в разделе "Пользователи". В данный 
момент регистрация новых пользователей возможна только через панель управления. 
Каждому пользователю необходимо назначить имя, пароль и его права. Есть 4 типа 
пользователей: администратор, модератор, пользователь и не активированный 
(или заблокированный) пользователь.

В Cafe CMS предусмотрено 4 уровня доступа:
0 - не активированный пользователь: этот уровень доступа запрещает авторизацию
на сайте;
1 - администратор: самый высокий уровень доступа, разрешено все;
2 - модератор: для этого уровня есть возможность работать в панели управления,
доступны все функции, кроме возможности удаления записей из базы данных;
3 - пользователь: обычный пользователь сайта, без доступа к панели управления.


 
=== Погода ===

Данные для модуля "Погода" предоставляет сервис openweathermap.org с помощью 
своего API. Для получения данных необходим API-ключ, который можно получить 
в личном кабинет после регистрации на сайте сервиса. Ссылка для регистрации 
http://home.openweathermap.org/users/sign_up.

После получения ключа, вы можете добавить необходимое количество городов. Для
этого перейдите в модуль "Погода" в панели управления, нажмите кнопку "Добавить
город" и заполните форму. В поле id-города необходимо написать номер
города. id можно посмотреть в файле на сервере openweathermap.org.
Ссылка на файл http://bulk.openweathermap.org/sample/city.list.json.gz. Размер 
файла в архивированном виде около 4 МБ, после распаковки около 19 МБ. 
Распакуйте скачанный архив, внутри будет json-файл. Откройте его и поиском 
найдите нужный вам город. Названия городов указаны по-английски. Например,
Moscow. Учтите, что в списке может быть несколько городов с одинаковым именем. 
Для точного определения в строке с городом так же указана страна. Например,
"country":"RU".

В поле ниже укажите частоту оновления погодных данных. Запросы будут 
кэшироваться в базе данных. При показе погодных данных, сначала осуществляется
проверка "свежести" кэша, если прошло времени больше, чем указано, будет сделан
новый запрос для получения погодных данных с openweathermap.org. В противном
случае, данные берутся из кэша.



=== Расписания ===
Данные для модуля "Расписниния" предоставляет сервис raspisanie.yandex.ru 
с помощью своего API. Для получения данных необходим API-ключ, который можно 
получить в кабинете разработчика https://developer.tech.yandex.ru.






=== Статистика ===

Для сбора статистики в модуле "Статистика" необходимо разместить код счетчика. 
Для этого нажмите на ссылку "Добавить счетчик" и заполните все поля.



=== Журнал действий ===
"Журнал действий" отображает все действия совершенные пользователями в панели
управления. По умолчанию журнал не ведется. В настройках журнала можно уровень
детализации журнала.



=== Настройки ===
Здесь собраны все основные настройки сайта. Сейчас их не много: название сайта, 
уровень детализации журнала и шаблон сайта.
